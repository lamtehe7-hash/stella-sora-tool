# Stella Sora Tool (SST) — ユーザーガイド

[Tiếng Việt](huong-dan-su-dung.md) | [English](user-guide.md) | **日本語**

> インストール → 設定 → 全機能まで網羅した完全マニュアル。
> **Android エミュレータ**(MuMu Player 推奨)上で動く **Stella Sora(EN/グローバル版)** 用です。
> スクリーンショットは実際のゲーム画面(アカウント名はぼかし済み)。

> ⚠️ **対応は EN(グローバル)版クライアントのみ。** 画面認識テンプレートが EN 版 UI 基準のため、
> **日本版クライアントでは動作しません**。ゲーム内のボタン名は EN 版の英語表記のまま記載します。

> ⚠️ 自動化はゲームの利用規約(ToS)に違反する可能性があります → **お試しはサブアカウント推奨**、
> ご利用は自己責任で。[README](../README_EN.md) の「Disclaimer & Terms of Use」を必ずお読みください。

## A. インストールと準備

### 1. 動作要件

| 項目 | 要件 |
|---|---|
| エミュレータ | MuMu Player(推奨)、または LDPlayer/BlueStacks — **ADB** が使えること |
| 解像度 | **必ず `1280 × 720`**(横 × 縦 — ゲームは横画面)。MuMu の「Android Device」パネルにこの数値が表示されます |
| ゲーム | **Stella Sora(EN 版)**、ログイン済みで Home 画面にいること |
| PC | Windows 10/11 |

> ⚠️ **解像度は正確に 1280×720 である必要があります。** ツールはこのサイズで切り出した
> テンプレート画像で画面認識しており、**自動スケーリングしません**。他の設定
> (1080×1920、1600×900 など)ではエラーで停止します。
> (デスクトップ上で MuMu の**ウィンドウ**を拡大縮小するのは無害です — 自由にズームしてください。)

### 2. 実行方法 — 2 通り

**方法 1: ポータブル版(Python 不要)**
1. Releases ページから最新の `StellaSoraTool-vX.Y.Z-win64.zip` をダウンロード。
2. フォルダごと解凍 — `app.exe`、`_internal/`、`assets/` を同じ場所に保つこと。
3. **`app.exe`** を実行。

**方法 2: ソースから実行(開発者向け)**
```powershell
python -m venv venv
venv\Scripts\pip install -r requirements.txt
python app.py        # デスクトップアプリ(推奨)
```

### 3. 初回設定(必須)

ツールを開く → **設定**ページで以下を入力:

| 項目 | 意味 | 例 |
|---|---|---|
| **エミュレータのシリアル** | エミュレータの ADB アドレス | `127.0.0.1:16384`(MuMu) |
| **adb.exe のパス** | エミュレータ付属の adb | `E:\MuMuPlayerGlobal\nx_device\12.0\shell\adb.exe` |
| **デイリーリセット(UTC)** | ゲームのリセット時刻(Yostar EN 標準 `11:00` UTC = 日本時間 20:00) | `11:00` |
| **Cleanup 時にゲームを閉じる** | 完了後にゲームを終了するか | オフ(Home に置いたまま) |

> **シリアル**の調べ方: エミュレータを起動した状態でターミナルから `adb devices` を実行すると
> 一覧表示されます(例 `127.0.0.1:16384`)。MuMu Player Global は通常ポート **16384** です。

保存すると `app.exe` の隣に `config/stella.json` が作成されます。

## B. ツールの画面構成

ツールの起動方法は **4 通り**:

| コマンド | モード |
|---|---|
| `app.exe` / `python app.py` | **デスクトップアプリ**(WebView2 ウィンドウ)— 推奨 |
| `python gui.py` | Web UI(`http://localhost:22270`) |
| `python sst.py` | CLI — 24 時間スケジューラループ |
| `python sst.py <Task>` | CLI — タスクを 1 つ実行して終了 |

**デスクトップアプリのレイアウト:**
- **左レール**: Home · sst(ダッシュボード)· 設定。
- **Home**: **言語 VI/EN** と**ライト/ダークテーマ**の切替。
- **ダッシュボード(sst)**:
  - **Start**(自動スケジューラ開始)/ **Stop**(実行中のタスクを即座に中断して停止)。
  - タスク一覧: 各タスクの**オン/オフ**、**今すぐ実行**ボタン(即座に実行予定にする)。
  - ステータスカード: Scheduler / 実行中 / 準備完了 / 待機中。
  - **ログ**パネル(Auto Scroll)— 動作をリアルタイムで確認。
- 一部のタスクには**専用の設定ページ**があります(Ascension、Bounty Trial、Event、
  Event First Clear、Heartlink)— [セクション D](#d-各タスクの詳細) 参照。

## C. ゲームのメイン画面(ハブ)

すべてのタスクは **Home** 画面から始まります。主なボタン(番号付き):

![番号付き Home 画面](images/home_annotated.png)

| # | ボタン | 関連タスク |
|---|---|---|
| 1 | **Mail**(封筒、右上) | `Mail` |
| 2 | **Missions** | `DailyReward` |
| 3 | **Shop**(左のアイコン) | `Shop` |
| 4 | **Commission** | `Dispatch` |
| 5 | **Grant** | `Grant` |
| 6 | **イベントバナー** | `EventDaily`, `EventFirstClear` |
| 7 | **Go**(バン、右下) | `BountyTrial`, `Ascension`(ここから入る) |
| 8 | **メニュー**(☰、最右上) | Daily Check-in、ゲーム内設定 |

右上のアイコン群には番号なしのものもあります: **Heartlink**(吹き出し)→ `Heartlink`、
**Purchase**(カート)→ `PurchaseGift`、**フレンド**アイコン(人物 2 人、Mail の隣)→ `FriendGift`。

> ナビゲーションの注意(重要): このゲームでは **Android の戻るキーは効きません** —
> 「戻る」操作はすべてゲーム内ボタン(サブ画面の 🏠 ボタンや画面ごとの退出ボタン)で行います。

## D. 各タスクの詳細

デフォルトの実行順:
`Login → Mail → Dispatch → Shop → PurchaseGift → BountyTrial → Ascension → EventDaily → EventFirstClear → Grant → Heartlink → FriendGift → DailyReward → Cleanup`

**デフォルトでオン**: Login, Mail, Dispatch, Shop, PurchaseGift, Grant, FriendGift, DailyReward, Cleanup。
**デフォルトでオフ**(必要な時にオンに): **BountyTrial, Ascension, EventDaily, EventFirstClear,
Heartlink** — リソース/チケットを消費するか、イベントごとの設定が必要なためです。

---

### 1. `Login` — ログインして Home へ
ゲームを起動し(未起動なら)、起動時のポップアップを抜けて Home に到達します。回線が切れた時の
**「Network Error」**ダイアログも処理(Retry/Start を押す)。他の全タスクの土台です。

### 2. `Mail` — メール受取
**Mail**(ボタン 1)を開き → **Claim All** で添付アイテムを全受取。受取済みは **"Claimed"** 表示。

![Mail 画面](images/mail.png)

### 3. `Dispatch` — 依頼(Commission)
**Commission**(ボタン 4)を開きます。仕事は 2 つ:
- **帰還した部隊の受取**: Claim All → 「Commission Complete!」ポップアップを **Back** で閉じる。
- **再派遣**: 空きスロットを **4/4** まで埋める — 各依頼 → **Quick Select**(条件に合う部隊を
  自動選択)→ **20h**(報酬最大)→ **Accept**。Trekker が尽きたら停止。

![Commission 画面](images/commission.png)

> 部隊は約 20 時間出発したままなので、多くの実行は「no-op」(まだ誰も帰っていない)になります —
> 無害です。Trekker 不足時はスキップして自動で再スケジュールします。

### 4. `Shop` — ショップの無料アイテム
**Shop**(ボタン 3)を開き → **無料/デイリー**アイテムを受取。受取済みは "Claimed" 表示(スキップ)。

![Shop 画面](images/shop.png)

### 5. `PurchaseGift` — 無料の Daily Gift
**Purchase** 画面(右上のアイコン群)を開き、無料の **Daily Gift** を受け取ります — ただし
赤い通知ドットが出ている時のみ。何もなければ画面にすら入りません。

### 6. `BountyTrial` — Vigor 消費(Quick Battle 掃討)· *デフォルトオフ*
**Go**(ボタン 7)→ **Bounty** → Trial を選択 → **Quick Battle** で Vigor の許す限り掃討します。

**専用設定ページ**(Bounty Trial):
| 項目 | 意味 |
|---|---|
| **Trial の種類** | `Basic`(素材)· `Tier-up` · `Skill` · `Emblem` |
| **難易度** | `0` = ゲームの記憶のまま; `1–6` = その段階を強制(掃討にはクリア済みであること) |

> ⚠️ このタスクは **Vigor を消費**します — Vigor を Trial に注ぎたい時にオンにしてください。

### 7. `Ascension` — Monolith 周回 · *デフォルトオフ*
**Go**(ボタン 7)→ **Ascension**。1 周につき **Monolith チケット 1 枚**を消費(Vigor 不要)。
カード選択、道中ショップでの購入、強化、Record 保存まで全自動。最も設定項目が多いタスクです。

**専用設定ページ**(Ascension)— 主なグループ:
| グループ | 代表的な項目 |
|---|---|
| チケットと周回数 | **1 回あたりの周回数**(チケット切れで自動停止) |
| マップ | ゲームの前回マップを維持 / Currents·Dust·Storm を強制(farming は Misstep 回避) |
| 難易度 | `0` = クリア済み最高段階を自動選択(推奨)/ 2–8 を強制 |
| 部隊 | `0` = そのまま / 部隊 N を強制 |
| プリセット | 「Preset not set」時: 警告 / スキップ / **エラーで停止**(デフォルト) |
| カード選択 | レベル上昇が最大(デフォルト)/ Super Rare 優先 / 一番左 |
| ショップ | Melody は必要時のみ購入 · 強化コインの閾値/確保 · 最終部屋の棚リフレッシュ |
| 周回 | Brief モード · Record 保存 · **Weekly Limit 満杯時はスキップ** · 最大周回時間 |

> デフォルト値はコミュニティのメタ(POWER — 強い Record)に沿っています。理解している項目だけ
> 変更してください。戦略の詳細: `docs/ascension-strategy.md`。

### 8. `EventDaily` — イベント掃討 · *デフォルトオフ*
Home の**イベントバナー**(ボタン 6)→ Battle Stage → **Quick Battle** で掃討; その後
赤ドットがあれば Event Mission の報酬も受取。

**専用設定ページ**(Event):
| 項目 | 意味 |
|---|---|
| **ステージ** | 空欄 = 最高ステージ; `W-N`(例 `1-12`)で特定ステージを指定 |
| **戦闘回数** | `0` = Vigor の許す限り; `N` = ちょうど N 回 |

> ⚠️ イベントは開催ごとに入れ替わります: **新イベントごとにバナーの再クロップ**
> (`assets/en/event/EVENT_BANNER.png`)とステージ確認が必要です。バナーが見つからない場合、
> ツールは安全にスキップします(間違ったものを実行することはありません)。

### 9. `EventFirstClear` — イベント初回クリア · *デフォルトオフ*
イベントの Battle Stage リストを巡回し、**グレーの星が残っているステージ**(未クリア)を
自動で戦います: ステージに入る → **Deploy** → **Auto-Battle** をオン → 勝利を待つ → 次へ。
クリア可能なものがなくなればきれいに停止(時限ロック中のステージはスキップ)。サイドバーの
**Event** グループに専用設定ページがあります。

### 10. `Grant` — Startup Grant 受取 · *デフォルトオン*
**Grant**(ボタン 5)→ **Startup Grant** 画面。タブが 2 つ(**Grant Milestone** と
**Company Goal**)あり、それぞれに赤ドットが付きます。ツールは **Company Goal を先に**受取
(進捗で Grant Tier が上がり Milestone 報酬が解放される)、その後両方 **Claim All**。
無料報酬なのでリスクなし。

![Startup Grant 画面](images/grant.png)

### 11. `Heartlink` — デートとギフト(Affinity)· *デフォルトオフ*
**Heartlink**(吹き出しアイコン、右上)を開きます。サブタスクは 2 つで、Heartlink 設定ページに
それぞれカードがあります:

- **Invite**(デート、**1 日 5 回まで**): **Invite** ボタンがまだティール色のキャラを選択
  (グレーの "Invited today" = 実施済み)、最初のデート場所を選択、**Skip** で早送り、最後に
  **Send Gift**(獲得 Affinity が 2 倍)または Leave。日次上限に達したら停止。
- **Mail / Delivery Service**(ギフト、**アカウント全体で 1 日 10 個**): Mail タブを開き、
  対象キャラを選択、ギフトを 1 つずつ送信。Affinity バーが動かなくなった瞬間(= 上限到達)に停止。

| カード | 項目 |
|---|---|
| Invite | 回数(デフォルト 5)· デート終了時に Send Gift · 名前指定の優先ターゲット(お気に入りを先にデート; お気に入りごとにポートレート画像の登録が必要、なければグリッド順) |
| Mail | 1 日のギフト数(デフォルト 10)· キャラごとの行(名前 + 個数、合計 ≤ 10)· または「常に一番上のキャラ」チェックボックス |

### 12. `FriendGift` — フレンドとスタミナ交換 · *デフォルトオン*
**フレンド**画面(人物 2 人のアイコン、右上)→ **Acquire All**(フレンドから届いたスタミナを
受取)+ **Gift All**(こちらからも送る)。その日の交換がカウントされるよう DailyReward の
直前に実行されます。

### 13. `DailyReward` — デイリーミッション
**Missions**(ボタン 2)→ Daily タブ → ミッションと活動ポイントのマイルストーン
(例: Stellanite Dust ×100)を **Claim All**。

![Missions 画面](images/missions.png)

### 14. `Cleanup` — 後片付けと終了
ゲームを Home に戻します。設定で **「Cleanup 時にゲームを閉じる」** がオンなら完全に終了します。

### (おまけ)Daily Check-in
**メニュー**(ボタン 8)→ **Daily Check-in** にあります。メニューにはゲーム内 **Settings**
(解像度変更)や Redeem Code などもあります。

![ゲームメニュー](images/menu.png)

## E. 自動スケジューラ

- 各タスクは**オン/オフ**スイッチと**「次回実行」**時刻(`next_run`)を持ちます。
- **Start** を押す → 期限が来たタスクを実行し、完了したタスクは**自分で次回を予約**:
  - デイリータスク → 次の**デイリーリセット**へ。
  - リソース不足のタスク(部隊が帰っていない Dispatch、Vigor 不足…)→ **数分〜数時間後**に再試行。
- **24 時間つけっぱなし**で運用可能: ループし、時間どおりに実行し、エラーからも自動復帰
  (ログ + スクリーンショットを保存)。
- タスクを手動で 1 つ実行: **今すぐ実行**(GUI)または `python sst.py <Task>`(CLI)。

> **Stop** を押すと**実行中のタスクを即座に中断**(次のスクリーンショット/クリック時点 —
> 最大でも数秒)してからスケジューラを停止します。中断されたタスクに**ペナルティはありません** —
> 次の Start でまた最初から実行されるだけで、ゲーム側も無傷です(ツールがボタンを押すのを
> やめるだけ; ゲーム内の中途状態はそのまま残ります)。アプリ/exe をそのまま閉じると操作の途中で
> プロセスが殺されます → 先に Stop を押すのがおすすめです。

## F. トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| 解像度エラー / 即停止 | エミュレータが **1280×720** でない → エミュレータの設定で修正 |
| 「`<serial>` に接続できない」 | **シリアル**か **adb.exe のパス**が間違い; エミュレータ未起動 → 設定を再確認 |
| ボタン誤認識 / 見知らぬ画面で停止 | ゲームの **UI アップデート**でテンプレート画像が破損 → アセットの再クロップが必要 |
| タスクは動くが「no-op」続き | リソースが揃っていない Dispatch/Bounty では正常 — 自動で再スケジュールされます |
| イベントが動かない | 新イベントのバナー**未再クロップ** / ステージ未設定 → EventDaily の項参照 |
| 別の PC で exe を実行 | その PC のエミュレータに合わせて**シリアル + adb パス**を更新 |

> ログとエラースクリーンショットは `app.exe` の隣の `log/` フォルダに自動保存されます —
> サポートを求める時に添付してください。
